# 铁矿石 CTA V1.4 激进收益版设计

## 目标

在 V1.3 已达到 387.34% 总收益、21.07% 年化、18.13% 最大回撤的基础上，向原始 V1 的收益能力靠近，同时避免恢复原始 V1 约 129.8% 策略波动率和 90% 级别最大回撤。V1.4 采用用户选择的 C 档风险预算：目标年化波动率 30%、最大保证金占用 60%、最大杠杆 3.5。

## 已验证的诊断

- 原始 V1 的超高收益主要来自极高名义暴露，不等同于更强的信号 alpha；其最大回撤和波动率不可接受。
- V1.3 相比 V1.1 已把总收益提升到 387.34%，年化 21.07%，但策略波动率达到 18.0%，夏普从 1.016 降到 0.946，说明简单整体放大仓位会牺牲风险收益效率。
- 因此 V1.4 的新增暴露只在趋势质量较高且波动未失控时触发，弱趋势、高波动、回撤阶段不获得额外杠杆。

## 设计

### 信号层

完全保留 V1.3/V1.1 的严格趋势信号：20 日快线、60 日慢线、10 日慢线斜率、连续 2 日确认、0.2% 价格缓冲、0.2% 均线间距、0.05% 最小斜率。V1.4 不新增宽松入场、不使用 V1.2 的 moderate/strong 趋势分类来过滤信号。

### 仓位层

基础合约数先由年化波动率、账户权益、可用现金、保证金和杠杆上限共同计算：

```text
基础仓位 = min(目标波动率 / 实现波动率, 最大杠杆, 最大保证金占用 / 保证金率)
           × 账户权益 / (合约价格 × 合约乘数)
```

之后依次乘以回撤乘数和趋势质量乘数。趋势质量乘数只允许 1.00 或 1.25：

- `close >= ma_fast * 1.01`
- `ma_fast >= ma_slow * 1.01`
- `slow_slope >= 0.001`
- `realized_vol <= target_annual_vol * 1.25`

四个条件同时满足时为 1.25，否则为 1.00。这样额外风险集中在趋势确认充分且波动仍可控的阶段。

### 回撤层

回撤相对历史高水位按以下规则缩放新开仓：

| 当前回撤 | 新开仓风险乘数 |
|---|---:|
| `< 10%` | 1.00 |
| `10% - <15%` | 0.90 |
| `15% - <20%` | 0.75 |
| `20% - <25%` | 0.50 |
| `>=25%` | 0.00 |

回撤乘数为 0 时停止新开仓，但不因回撤阈值单独强平已有仓位；已有仓位仍按反向信号、换月和 ATR 止损退出。

### 执行与数据安全

- 所有行情使用 `end_date=signal_date`。
- 期货元数据使用带日期的 `get_all_securities(["futures"], date=signal_date)`。
- 换月始终先平旧合约；旧合约实际持仓未归零时不允许开新合约。
- 保持 `ALLOW_SHORT=False`，默认只做多。
- 输出为不依赖 `reports` 包和 `from __future__` 的单文件聚宽脚本。

## 验证标准

- 新增纯函数测试覆盖严格信号、趋势质量乘数、四档回撤边界、激进仓位计算、ATR/波动率、近月选择和先平后开保护。
- 聚焦测试、项目全量测试、Python 编译、静态审计和测试替身运行时冒烟全部通过。
- 桌面直贴副本与工作区脚本 SHA256 一致。
- 不宣称回测收益改善，必须由用户在聚宽同一时间区间运行 V1.3/V1.4 对比确认。
