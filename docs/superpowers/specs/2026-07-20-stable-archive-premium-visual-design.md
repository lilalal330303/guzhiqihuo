# 稳定策略档案库质感化视觉设计

## 目标

在不改变四个策略的名称、数据、切换逻辑、研究结论和页面信息架构的前提下，提升稳定策略档案库的视觉质感，让它更像一份成熟的量化研究终端档案，而不是普通静态说明页。

## 视觉方向

- 主色使用深海墨蓝、石墨黑和低饱和冰蓝；以少量香槟金作为证据、稳定状态和关键数值的强调色。
- Hero 使用多层径向光晕、精细网格、低频光带和轻量噪点感，形成有深度的研究终端背景；所有装饰均使用 CSS，避免远程资源与加载失败。
- 内容区使用偏冷的云雾白底、半透明白色卡片和细金属边框；保留足够留白，保证数据可读性。
- 策略切换器采用胶囊标签与连续高光轨迹，选中态明显但不喧宾夺主；切换策略时保留现有数据驱动逻辑，并增强卡片的短暂入场反馈。
- 指标、参数、因子和证据卡片统一圆角、边框、阴影和 hover/focus 语言，形成一套稳定的视觉组件系统。
- 移动端继续单列布局，减少装饰层强度；`prefers-reduced-motion: reduce` 时关闭连续动画，只保留即时状态变化。

## 页面层级

1. 首屏：品牌、档案库标题、策略切换、稳定状态、主标题、验证区间、基准和核心收益卡。
2. 证据快照：指标卡以第一视觉优先级呈现，增加细微的数值强调和数据轨迹装饰。
3. 方法与信号：保持现有因子、MinMax 公式和执行流程内容，只升级容器层次和分隔关系。
4. 参数与回测证据：用冷白纸张感承载表格，保证长周期数据的严肃性。
5. 运行边界：保持免责声明与适用范围文案，不添加收益承诺或下载入口。

## 实现边界

- 修改范围限定为 `strategy-024.css`，仅在确有必要时补充 `strategy-024.html` 的语义类名；不修改策略对象中的内容和数字。
- 不引入框架、字体、图标库、远程图片、外部 CDN、canvas 或音频。
- 保留现有四个 `data-strategy` 键：`value`、`wufu`、`fuxing`、`smallcap`。
- 保留现有策略切换、锚点导航、移动端单列和无 JavaScript 提示。
- 不恢复已删除的资料来源、下载入口、返回研究主页按钮或 `024` 可见编号。

## 动效与可访问性

- Hero 网格和光晕以 20–32 秒低频循环，避免干扰阅读。
- 卡片 hover 只使用轻微位移与边框亮度变化，不改变布局尺寸。
- 策略切换触发内容淡入、指标上移和高光短脉冲，动画时间控制在 500ms 内。
- 所有交互控件保留可见 `:focus-visible` 状态；正文和指标保持 WCAG AA 级别的对比度目标。
- `prefers-reduced-motion` 下取消连续背景动画和入场动画，内容仍立即可见。

## 验收标准

- 桌面宽屏首屏有明显的深度、层次和研究终端质感，且标题、切换器和核心指标在 3 秒内可识别。
- 四个策略切换后标题、指标、参数、证据和主题色仍与原始数据一致。
- 手机宽度下无横向滚动，切换器可换行，表格和卡片可读。
- 静态检查确认未增加外部资源依赖，且策略内容和 `data-strategy` 键未被改动。
- 构建/静态预览通过后，再把同一份页面发布到 GitHub Pages 与现有国内 CDN。
