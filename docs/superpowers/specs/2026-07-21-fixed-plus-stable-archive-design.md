# 固收+稳定策略档案接入设计

## 目标

把量化研究目录中的 `joinquant_fixed_plus_stable_current.py` 作为“固收+策略”接入现有稳定策略档案库，并沿用已经上线的单页切换模板。策略逻辑、交易代码和参数不在本次任务中修改。

## 方案

- 在 `strategy-024.html` 的策略切换器中新增 `fixedplus` 标签“固收+策略”。
- 在同一份策略数据注册表中新增完整档案：定位、资产配置、风险预算、调仓流程、参数、执行边界、证据口径和运行注意事项。
- 使用 `fixedplus` 主题色，保持现有模板的卡片、指标、流程、参数表、证据表和注意事项布局不变。
- 将稳定版聚宽脚本复制到档案库根目录，作为归档资产保存；页面不增加下载入口。
- 不把目标波动率、配置比例等参数冒充为收益回测结果。当前缺少随仓库发布的固收+长周期收益快照，因此页面明确标注“参数锁定版，收益快照待补录”，只展示可以从脚本直接核对的配置与风控事实。
- 更新入口页和档案页的策略数量描述，使其与新增第五套策略一致。

## 资产配置与执行事实

稳定版脚本的目标权重为：511010.XSHG 60%、511990.XSHG 5%、518880.XSHG 10%、510880.XSHG 12.5%、513100.XSHG 12.5%。脚本以 511010.XSHG 为基准，按 60 日波动率倒数对风险资产做风险预算，组合目标年化波动率为 6%，每 20 个交易日检查一次再平衡，保留 2% 现金缓冲，使用 100 份整手和涨跌停/停牌保护。

## 验证与发布

1. 校验新页面仍能通过 `#fixedplus` 直接打开，并检查五个策略标签、页面文案和脚本归档文件。
2. 运行固收+脚本导出相关测试及 `git diff --check`。
3. 提交到 `main` 并推送 GitHub。
4. 等待 `Deploy stable strategy archive to CloudBase` 完成，确认登录、静态站点部署步骤成功。
5. 对腾讯云公开根地址和 `strategy-024.html#fixedplus` 发起 HTTP 验证。

## 不在本次范围

- 不修改固收+策略本身的交易逻辑。
- 不把 selected/aggressive 两个固收+路线混入稳定档案。
- 不重做已有四个策略的内容和指标。
