# 大小盘反复横跳 V3.1 收益优先型迭代设计

## 1. 文档状态

- 状态：已确认设计，待实施计划
- 日期：2026-07-20
- 目标版本：V3.1 参数实验版
- 基线脚本：`reports/joinquant_size_style_rotation_v30_original_replica.py`
- 默认运行模式：`ORIGINAL_REPLICA`

本设计只定义下一轮实验边界，不直接修改 V3.0 基线脚本。实施前必须先通过基线复核，再按单变量顺序生成实验脚本或参数版本。

## 2. 背景与基线

用户已将策略回退到原始版本，当前以 JoinQuant 回测截图作为收益优先型基线。基线指标如下：

| 指标 | 基线 |
|---|---:|
| 策略总收益 | 832.02% |
| 策略年化收益 | 42.23% |
| 超额收益 | 639.49% |
| 基准收益 | 26.04% |
| Alpha | 0.385 |
| Beta | 0.776 |
| 夏普比率 | 1.936 |
| 胜率 | 0.589 |
| 最大回撤 | 25.55% |
| 盈利次数 / 亏损次数 | 155 / 108 |
| 日均超额收益 | 0.14% |
| 超额最大回撤 | 26.13% |
| 超额收益夏普比率 | 1.398 |
| 策略波动率 / 基准波动率 | 0.281 / 0.202 |

这条曲线已经具有较强的收益特征。因此本轮不以“增加防守模块”为主，而以确认收益来源、减少无效试错和寻找低扰动参数为主。

## 3. 目标与非目标

### 3.1 目标

在保持原始收益结构的前提下，寻找能够保留主要收益、同时允许回撤小幅上升的稳定参数组合：

1. 先确认 V3.0 能复现原始基线，而不是把接口或执行错误当作策略结果。
2. 识别收益主要来自风格切换、分支选股还是持仓集中度。
3. 将每次实验限制为一个主要变量，确保结果可解释、可回退。
4. 为最终聚宽脚本保留原始默认参数，实验参数通过顶部配置切换。

### 3.2 非目标

本轮不引入以下已验证为低质量或高风险的路径：

- 趋势确认、双窗口趋势过滤；
- 波动率归一化；
- `risk-off` 现金防守；
- 风格滞后确认、迟滞区间；
- 候选缓冲、强制保留旧持仓；
- 市场保护阀、指数过滤；
- winsorization 或其他收益截尾；
- 严格时点口径作为默认策略；
- 通过 `pivot_table(sort=...)` 等不兼容接口改变策略逻辑。

这些内容可以作为历史失败记录保留，但不得重新混入 V3.1 的主实验链。

## 4. 总体设计原则

### 4.1 基线优先

V3.0 `ORIGINAL_REPLICA` 是唯一比较基准。任何实验必须先记录完整样本和三个时间窗口的结果，再与同一版本基线比较。

### 4.2 单变量优先

第一轮只改变一个参数。只有某个参数在多个窗口都通过门槛，才允许进入组合实验。禁止直接同时修改窗口、阈值、持仓数和交易成本。

### 4.3 收益优先、回撤设护栏

本轮优先保留年化收益和总收益，不要求每个候选方案都降低回撤。但如果回撤扩大到失控、夏普明显坍塌或收益只集中在单一时期，必须淘汰。

### 4.4 可观察、可回退

每次月度调仓记录：

- 风格分支和两个分支的 20 日横截面平均收益；
- 风格窗口与阈值；
- 目标股票列表；
- 调仓前后持仓；
- 卖出、买入、未成交订单；
- 每月换手和空仓月份；
- 候选列表为空或数据不可用的原因。

## 5. 分阶段迭代路径

### 阶段 0：基线复核

保持 V3.0 所有参数和执行设置不变，使用原始截图对应的日期、初始资金、基准和分钟频率运行。

复核内容：

- 是否能够产生交易；
- 每月风格分支是否有信号；
- SMALL / BIG 是否能够返回目标列表；
- 交易数量、换手、空仓月份是否合理；
- 收益曲线与原始截图是否处于同一数量级。

阶段 0 未通过时，停止优化，不进入参数实验。

### 阶段 1：风格信号参数实验

保持选股、持仓和调度全部不变，仅测试风格切换参数。

实验变量：

| 实验 | 参数 | 候选值 | 默认值 |
|---|---|---|---:|
| S1 | 风格阈值 | 1.10、1.15、1.20、1.25、1.30 | 1.20 |
| S2 | 风格收益窗口 | 10、20、40、60 个交易日 | 20 |

S1 完成后再进行 S2。若某个候选只在全样本有效、但在三个分段中有两个明显失效，不进入组合实验。

### 阶段 2：持仓集中度实验

固定阶段 1 选出的风格参数，只改变持仓数量：

- 3 只；
- 5 只；
- 8 只；
- 10 只。

先测试 SMALL 与 BIG 共用一个持仓数量，再测试分支独立持仓数量。分支独立参数只有在共用参数实验完成后才允许引入。

### 阶段 3：分支组合结构实验

固定风格参数和最终持仓数量，只研究分支内部的组合方式：

1. BIG 分支使用 3 只或 5 只；
2. SMALL 分支使用 5 只或 8 只；
3. 等权与轻度市值权重二选一做对比；
4. 保持原始候选排序，不引入新因子，不改变基础筛选条件。

该阶段的目的，是判断收益来自“持有哪一批股票”还是“持有多少只股票”，而不是重新发明选股模型。

### 阶段 4：交易成本与执行敏感性

在信号和组合结构确定后，单独测试：

- 零滑点基线；
- 低滑点情景；
- 压力成本情景；
- 原始调仓时间与一个替代调仓时间。

成本敏感性只用于评估策略可实施性，不得用更乐观的成本假设制造收益提升。

### 阶段 5：最终组合与样本外复核

仅将通过单变量门槛的参数组合成最终候选，并在以下区间分别运行：

- 2020—2021；
- 2022—2023；
- 2024—2026；
- 全样本。

最终只保留一套默认参数，其他方案作为对照记录，不在脚本中自动择优。

## 6. 验收门槛

### 6.1 基线复现门槛

由于 JoinQuant 数据接口、日期边界和执行环境可能产生轻微差异，V3.0 基线应满足：

- 总收益和年化收益与截图处于同一数量级；
- 最大回撤与截图方向一致；
- 有持续月度交易，而不是仅产生起始两笔订单；
- 风格信号和目标列表不可系统性为空。

若出现“无交易”“风格信号不可用”“候选列表不可用”持续重复，视为工程失败，不视为策略表现。

### 6.2 候选方案门槛

以 832.02% 总收益、42.23% 年化收益、25.55% 最大回撤和 1.936 夏普为基准，候选方案至少满足：

- 年化收益不低于约 40%；
- 总收益不低于约 750%；
- 最大回撤不超过 35%；
- 夏普不低于 1.40；
- 三个分段中至少两个分段没有出现收益结构崩塌；
- 交易次数或换手不能出现无法由参数解释的异常膨胀。

回撤小幅高于基线可以接受，但收益大幅下降的方案不接受，即使回撤更低也不进入主线。

## 7. 实验记录格式

每个实验至少记录：

- 实验编号、父版本和唯一变更参数；
- 日期范围、初始资金、基准、运行模式；
- 风格窗口、阈值、持仓数量、成本设置；
- 总收益、年化收益、最大回撤、夏普、胜率；
- 交易次数、换手、空仓月份；
- 三个分段指标；
- 目标列表差异和主要交易变化；
- 是否通过门槛；
- 下一步动作：保留、淘汰或进入组合实验。

## 8. V3.1 实施边界

V3.1 实验脚本应满足：

1. 顶部集中定义 `STYLE_WINDOW`、`STYLE_THRESHOLD`、`STOCK_NUM` 等参数；
2. 默认参数与 V3.0 完全一致；
3. 默认运行仍为 `ORIGINAL_REPLICA`；
4. 不改变既有 JoinQuant 兼容适配器；
5. 不引入 pandas 新版本专属参数；
6. 保留原始日志和诊断信息；
7. 用函数而不是复制多套策略实现参数切换；
8. 提供最小化的本地静态测试，验证默认参数、候选值和关键逻辑没有越界。
