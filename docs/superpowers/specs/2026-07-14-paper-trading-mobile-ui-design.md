# 模拟盘移动端 UI/UE 适配设计

日期：2026-07-14

## 目标

让模拟盘在 360–480px 手机宽度下具备接近原生金融应用的浏览体验，解决顶部导航竖排、账户卡片占屏过高、图表空间不足和宽表难以阅读的问题，同时保持现有桌面布局、策略隔离和数据口径不变。

## 设计方案

采用响应式双布局：桌面端继续使用顶部导航与左侧策略账户栏；宽度不超过 680px 时切换到移动端专用信息架构。

### 顶部区域

- 手机端顶部固定为单行工具栏，仅保留“量化研究 / 模拟盘”品牌、当前策略简称和主题切换按钮。
- 隐藏桌面顶部导航和冗长快照时间，避免中文被压成竖排。
- 工具栏高度控制在 56px，并适配手机安全区域。

### 主导航

- 总览、持仓、订单、日志改为底部固定四栏导航。
- 当前页面使用高对比色、图标与文字共同表达选中状态。
- 底部导航预留 `safe-area-inset-bottom`，页面正文增加相应底部留白，避免遮挡内容。

### 策略账户选择

- 左侧策略栏在手机端变为顶部横向滚动的紧凑账户选择器。
- 卡片显示策略名称、整数权益和持仓数量；单张卡片宽度约为视口的 72%，支持横滑和吸附。
- 当前策略使用边框与浅色背景强调，不使用深色整块底板，以保证亮色主题可读性。

### 页面标题与快照

- 标题、描述和快照状态改为纵向紧凑排列。
- 快照默认展示“行情截至”和“15:05盘后快照”，导出时间作为次级信息。
- 减少上下留白，保证首屏可同时看到策略名称、状态与主要指标。

### 指标与图表

- KPI 保持两列，使用紧凑内边距、响应式字号和最小宽度保护。
- 权益图容器在手机端使用约 260px 高度，SVG 保持完整坐标轴和触摸提示。
- 周期按钮占满可用宽度，触控区域不小于 44px。

### 持仓、订单和日志

- 桌面端继续使用表格。
- 手机端表格容器允许横向滚动，并增加明显的“左右滑动查看完整信息”提示；首列保持可辨识，不压缩成竖排。
- 策略页标签使用横向滚动，选中状态保持浅色高对比，不改变内容底板颜色。
- 日志和执行活动使用单列卡片，时间与状态分层显示。

## 交互与可访问性

- 所有导航、周期、主题和标签按钮保持键盘焦点样式。
- 手机触控目标最小高度 44px。
- 横向滚动区域启用惯性滚动和滚动吸附。
- 支持系统减少动画偏好。
- 亮色为默认主题，深色切换继续记忆。

## 数据与架构边界

- 不修改 V7K、V12D 策略代码、参数、账户状态或快照结构。
- 不新增后端依赖；主要修改 `docs/paper-trading/styles.css`，仅在必要时调整 `app.js` 的移动导航标记和辅助提示。
- 所有页面继续共用相同静态资源和动态快照。

## 验收标准

- 360px、390px、430px 和 480px 宽度下，顶部导航不出现竖排文字或横向页面溢出。
- 首屏可见当前策略、行情状态和至少一组主要指标。
- 底部四栏导航可跳转并正确显示当前页面。
- 策略账户卡片可横滑选择，当前策略文字在亮暗主题下均清晰。
- KPI、周期按钮和图表不超出卡片边界。
- 表格可横滑查看，页面主体本身不产生横向滚动。
- 桌面端顶部导航、左侧栏和内容布局保持现状。
- 静态页面回归测试和本地 HTTP 验证通过，修改同步到 GitHub Pages。
