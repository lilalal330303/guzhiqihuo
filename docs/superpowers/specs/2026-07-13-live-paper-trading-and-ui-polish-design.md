# 实盘行情模拟盘与页面优化设计

## 目标

让 V7K 与 V12D 模拟盘按交易日的真实分钟行情自动推进，并优化静态模拟盘页面的中文表达、持仓汇总、固定侧栏和长数据布局。核心策略信号、执行时点与参数保持冻结。

## 运行机制

- 盘中运行器仅在交易日、策略允许分钟执行；先获取与校验最新 ETF 分钟行情，再调用既有模拟盘执行器。
- 收盘补齐当日分钟数据并进行审计复核；数据完整时导出 GitHub Pages 快照。
- 本地分钟数据为审计主源；腾讯实时行情作为盘中校验/补充来源。
- 行情缺失、非交易日或数据校验失败时，记录明确的阻塞原因，不创建虚假订单、成交或权益快照。
- 模拟盘页面明确展示“交易数据截至”和“快照生成时间”；两者不可混淆。

## 页面优化

- 用户可见文案、状态和列名一律中文；审计策略标识保留在详情中。
- 总览与全局持仓按标的合并，展示合计数量、合计市值和策略归属；策略详情保持账户级明细。
- 左侧导航在桌面视图固定于可视区域，内容区独立滚动。
- 表格与长文本使用固定列宽、省略提示、详情展开和水平滚动容器；指标、图表、筛选器在窄屏自动换行。

## 数据边界

新增数据接入、调度和导出只写本地模拟盘审计记录与快照文件。任何新策略账户可由配置自动被运行器与导出器发现。GitHub Pages 仍为静态已发布快照，本地页面显示最新本地审计状态。

## 验证

测试真实分钟行情校验、非交易日跳过、缺失行情阻塞、成功推进、持仓合并及页面文案/布局契约。使用本地 HTTP 服务验证页面互动，并运行全量测试后再发布。
