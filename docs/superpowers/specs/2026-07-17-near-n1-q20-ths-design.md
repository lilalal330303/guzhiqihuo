# near_n1_q20 同花顺 SuperMind 复刻设计

## 目标

在 SuperMind/同花顺 Python 3.8 分钟回测环境中，复刻本地稳定版本 `near_n1_q20` 的可执行规则，并保留可审计日志，供平台回测结果与本地结果逐项对比。

## 复刻边界

- 选股池：399101（中证1000）成分股；过滤停牌、ST、退市标记、科创/北交所/创业板代码，以及上市时间不足 375 天的股票。
- 基础排名：按上一交易日可见的总市值升序取候选池，目标持仓数为 5。
- 临界优化：取排名前 `N+1`（N=5），按“80%市值排序分 + 20%质量排序分”选回 N 只，质量分由盈利、现金流、杠杆和收入/市值等字段构成。
- 数据降级：若同花顺环境没有可调用的财务因子接口，脚本不伪造财务值，而是记录 `QUALITY_FALLBACK` 并退化为纯市值临界排序；该回测只能称为“同花顺兼容降级版”，不能称为完整复刻。
- 执行：`handle_bar` 按分钟时间门控，09:05 生成目标、09:45 执行调仓、10:00 止损/清仓检查、14:00 涨停开板检查、15:10 记录；所有卖出遵守 `closeable_amount`（T+1）。
- 风控：固定止损 11%、市场跌幅止损 5%、成本保护、拥挤度仓位预算；ATR 和聚宽专属财务公告接口不在同花顺降级版中强行模拟。
- 交易：股票按 100 股整数倍下单，使用同花顺 `order_target_value`；佣金、印花税和滑点通过平台配置，脚本仅保留日志中的成本预算。

## 组件与数据流

1. `init(context)` 初始化参数、状态缓存、交易成本和订阅池。
2. `handle_bar(context, bar_dict)` 通过 `get_datetime()` 触发日内阶段，避免依赖可能不可用的 `run_daily`。
3. `build_stock_pool()` 拉取指数成分股和上一交易日行情；先做交易状态过滤，再做市值排序。
4. `quality_scores()` 尝试调用平台财务因子；接口不存在或返回空时，输出降级标记并返回中性质量分。
5. `select_near_n1_q20()` 在前 N+1 名中应用临界质量加权，写入 `NEAR_SIGNAL` 日志。
6. `rebalance()` 先卖后买，使用 `closeable_amount`、涨跌停和可用现金检查，写入 `THS_ORDER` 日志。
7. `risk_checks()` 在 10:00/14:00 及分钟循环中执行止损、涨停开板和市场仓位预算。

## 错误处理

- 任一数据接口失败时，保留当前持仓，不清空目标；日志记录接口名、日期和降级原因。
- 质量因子缺失只允许中性降级，不允许把未来数据或本地 CSV 注入回测。
- 证券代码统一转换为同花顺 `.SH/.SZ` 格式，同时保留原始代码用于日志。
- 每个交易日最多执行一次信号和一次调仓，防止 `handle_bar` 重复下单。

## 验收标准

- 本地 Python 3.8 语法编译通过，且不依赖 `jqdata`、`jqfactor`、`finance.run_query`。
- 静态检查确认包含 `init`、`handle_bar`、`before_trading`、`after_trading`，并包含 `NEAR_SIGNAL`、`THS_ORDER`、`QUALITY_FALLBACK` 日志标记。
- 你在同花顺页面编译运行后，需提供策略收益、最大回撤、Sharpe、日志和回测区间；本地报告只把该结果标记为“平台实跑”，不自行推断。
