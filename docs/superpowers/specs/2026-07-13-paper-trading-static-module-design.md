# 模拟盘静态前端模块设计

## 目标

将当前单一展示页扩展为可部署在 GitHub Pages 的模拟盘前端模块。它展示两套独立账户（七星五福 V7K、单五福 V12D）的本地模拟交易审计快照，并提供与真实交易页面一致的信息层级、图表坐标和浏览器端交互。

本模块只读取模拟盘审计结果；不得修改 V7K、V12D 的信号逻辑、执行时点或参数。

## 方案选择

选择“静态页面 + 导出 JSON 快照 + 原生 JavaScript”方案。

- 静态页面适合 GitHub Pages，无服务端和登录依赖。
- 快照导出器从本地 DuckDB 读取审计记录，消除当前页面写死数据的问题。
- 原生 SVG 图表提供金额、收益率、时间坐标与悬浮提示，不依赖外部 CDN。

不选择把 Streamlit 发布到 GitHub Pages：Streamlit 是本地实时运营入口，GitHub Pages 无法运行 Python 或连接本地 DuckDB。

## 信息架构

页面目录为 `docs/paper-trading/`：

- `index.html`：策略总览；账户卡、聚合指标、收益曲线和最近活动。
- `strategy.html?id=<strategy_id>`：单策略详情；收益/回撤图、持仓、订单、成交、信号与运行日志页签。
- `positions.html`：全部账户持仓，支持按策略筛选。
- `orders.html`：订单及成交流水，支持策略、状态和关键词筛选。
- `logs.html`：审计异常、运行记录与数据就绪状态，支持策略筛选。
- `styles.css`：共享的深色科技风布局、响应式规则和视觉语义。
- `app.js`：共享导航、快照加载、筛选、详情页签、SVG 图表和空态处理。
- `data/snapshot.json`：由导出器生成的只读页面数据。

页面使用查询参数选择策略，因而新增策略只需在账户配置与审计库中出现，导出的快照就会自动在总览、筛选器和详情页中出现，无需复制页面模板。

## 数据与边界

新增导出模块负责把 DuckDB 中的账户、权益曲线、持仓、订单、成交、最新意图、异常和就绪状态整理成 JSON。所有金额、数量、状态、时间均来自审计库；页面只格式化，不重新计算交易结果。

快照包含生成时间和“本地模拟盘审计快照”来源标识。GitHub Pages 展示的是发布时的静态快照；本地 Streamlit 页面继续作为实时运行和推进入口。

JSON 不含本地绝对路径、密钥或原始数据供应商凭证。订单、成交和日志只导出页面所需的最近有限记录，避免仓库持续膨胀。

## 交互与视觉

- 左侧策略列表/顶部导航均能跳转，并显示当前选中项。
- 图表拥有明确的 X 轴时间、Y 轴金额或收益率刻度；鼠标悬停显示最近数据点；可切换权益与回撤序列。
- 策略详情使用真实页签，在浏览器端切换对应数据表。
- 持仓、订单和日志提供浏览器端筛选；筛选后显示记录数和空态。
- 订单行可展开查看审计字段；无订单或无信号时明确提示，不伪造数据。
- 响应式布局在桌面和窄屏下保持导航、核心指标和表格可用。

视觉只借鉴模拟盘的账户—指标—图表—审计信息结构，不复制同花顺的品牌、图标、代码或登录内容。

## 错误处理

当 `snapshot.json` 不可读取、内容为空或策略参数无效时，页面显示可理解的错误状态和返回总览入口。所有表格对空集合显示空态。快照生成失败应返回非零状态，不覆盖已有有效快照。

## 验证

- Python 测试验证导出 JSON 的账户动态性、关键字段、时间序列和无写库副作用。
- 静态页面检查验证所有页面、共享资源和快照引用存在。
- 本地浏览器打开各页面，验证图表坐标、悬浮提示、页签、筛选、跳转和窄屏布局。
- 执行相关测试及全量测试；只提交本模块新增或修改的文件，再推送到 `master`。
