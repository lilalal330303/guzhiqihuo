# 大小盘反复横跳 V3.0：原始复刻优先设计

## 1. 目标

以用户提供的原始版 `39干积分-大小盘反复横跳V2.0.txt` 为回测行为基线，重新建立一个可以在聚宽中直接运行、能够逐月核对交易路径的独立脚本。

本轮首要目标是恢复原始版的信号、候选池、分支方向和调仓路径；只有在基线对齐后，才允许开展单变量优化。默认版本不再引入 V2.1/V2.2 已验证会显著改变结果的趋势确认、波动率风控、risk-off 空仓、候选缓冲、风格滞后和复杂保护逻辑。

## 2. 证据与根因判断

原始版与 V2.2 的差异中，以下两项会直接改变回测收益，而不是普通代码健壮性修复：

1. 原始版基本面查询使用 `get_fundamentals(..., date=None)` 或省略日期；V2.2 改为 `date=context.previous_date`。
2. 原始版风格信号使用 `get_index_stocks(index, context.current_dt)`；V2.2 改为 `date=context.previous_date` 的历史成分。

这两项改变可能显著减少幸存者偏差和未来信息，因此 V2.2 的收益不能直接与原始截图比较。它们应当成为独立实验变量，而不是混入一个默认版本。

已验证不作为本轮主线的路径：

| 路径 | 处理 |
| --- | --- |
| V2.1 指数趋势确认、波动率标准化、risk-off 空仓 | 跳过 |
| V2.1 风格分数、候选缓冲、最短持有期、切换滞后 | 跳过 |
| V2.2 默认严格历史成分与严格历史基本面 | 保留为 `STRICT_ASOF` 对照，不作为默认 |
| 收益截尾、市场均线保护、滑点改造 | 默认关闭，后续单独测试 |

截图中的原始版收益只作为用户提供的参考现象，不在本地规格中填入或推断具体收益数字。

## 3. 运行模式

脚本为一个独立文件，顶部提供：

```python
RUN_MODE = "ORIGINAL_REPLICA"
```

### 3.1 `ORIGINAL_REPLICA`（默认）

该模式优先复刻原始版行为：

- 基准 `000985.XSHG`；真实价格；原始订单成本；`FixedSlippage(0)`。
- `stock_num=5`；SMALL 价格上限 10 元。
- 09:05 准备持仓，月初第一个交易日 09:30 调仓，14:00 检查昨日涨停持仓，14:30 保留原始账户清理入口。
- 风格池使用 `399303.XSHE` 和 `399905.XSHE`；每只成分股使用截至昨日的 20 个日线收盘价计算首尾收益横截面均值。
- 保留原始有效分支方向：`mean_BIG / mean_SMALL > 1.2` 进入原始命名的 `BIG` 分支，否则进入 `SMALL` 分支。
- `SMALL` 保留原始全市场过滤、ROE/ROA 条件、低价条件、小市值排序、40 日最近涨停黑名单，并在黑名单过滤后取前 5 名。
- `BIG` 保留原始估值、盈利、利润率、收入增长和大市值排序条件，并保留原始查询实际使用宽池 `stocks` 的行为。
- 基本面查询的日期语义保持原始版，以便先对齐原始回测交易路径。
- 指数成分查询的日期语义保持原始版，以便先对齐原始回测交易路径。

该模式是研究/回测复刻模式，不宣称无未来数据。代码中必须用注释和日志明确标注这一点。

### 3.2 `STRICT_ASOF`（对照）

只改变两类日期语义：

- 基本面查询统一使用 `date=context.previous_date`。
- 指数成分统一使用 `date=context.previous_date`。

其余信号、候选、调仓、费用和时间设置必须与 `ORIGINAL_REPLICA` 相同。该模式用于判断收益差异是否主要来自数据时点，而不是用于替换默认基线。

## 4. 兼容层边界

兼容层只处理会让脚本在聚宽运行时报错、但不应改变策略含义的问题：

- 不使用旧版 pandas 不支持的 `pivot_table(sort=...)` 参数。
- 允许 `get_price(..., panel=False)` 返回长表或宽表时提取 `close`，但不引入新的截面过滤、截尾或风控。
- 下单接口仍使用 `order_target_value`，保留原始停牌、涨跌停失败时的安全日志。
- 数据获取异常时记录明确日志；如果无法生成信号或候选池，则保持现有持仓，不将异常转换为空仓。

兼容层不能改变：指数成分的日期语义、基本面的日期语义、20 日窗口、收益计算方式、阈值、候选排序和调仓频率。

## 5. 单变量优化路径

本轮输出的脚本只提供基线和严格历史对照，不把未经验证的 alpha 改动塞入默认路径。基线对齐后，按以下顺序开展后续实验：

1. **E1 数据时点实验**：仅切换 `ORIGINAL_REPLICA` 与 `STRICT_ASOF`，确认收益差异来自基本面/成分日期还是来自信号实现。
2. **E2 成本实验**：固定 E1 胜出的信号路径，只比较 `slippage=0 / 0.001 / 0.002`，不同时修改选股。
3. **E3 交易频率实验**：在信号和候选完全不变的前提下，比较月初调仓与固定月内第二调仓日；若没有跨阶段改善则淘汰。
4. **E4 候选池实验**：只比较 BIG 查询使用 `stocks` 与使用过滤后 `choice`，不加入新的财务因子。

任何实验必须在全样本、2020-2021、2022-2023、2024-2026 四个窗口中同时记录总收益、年化收益、最大回撤、交易次数、胜率、换手率和空目标次数。全样本单独改善不能进入默认版。

## 6. 日志与可核对性

每次月度调仓至少输出：

- 运行模式、两个风格均值和最终分支；
- SMALL/BIG 候选数量、最终目标列表；
- 当前持仓、卖出列表、买入列表和实际成交结果；
- 信号不可用、候选为空、订单失败的具体原因。

这样可以先核对“分支和目标股是否与原始版一致”，再讨论收益，而不是只看最终收益曲线。

## 7. 验证标准

### 本地验证

- 独立脚本可在空 `jqdata` 桩下被加载。
- 纯函数测试覆盖原始阈值方向、20 日首尾收益、SMALL 前 5 名、BIG 宽池、模式开关和目标合并。
- AST 检查确认两种模式的日期调用位置明确，且默认模式没有意外启用 V2.1/V2.2 风控。
- Python 编译和差异检查通过。

### 聚宽验证

1. 使用与原始截图一致的时间、资金、基准、费用和调仓设置。
2. 先运行 `ORIGINAL_REPLICA`，逐月比较分支、目标列表、交易次数和成交结果。
3. 只有基线路径对齐后，才运行 `STRICT_ASOF` 和 E1-E4。
4. 本地测试不声称聚宽云端收益，也不以截图数字填补未运行的回测结果。
