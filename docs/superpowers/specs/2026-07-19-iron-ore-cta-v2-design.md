# 铁矿石 CTA V2 设计说明

**日期：** 2026-07-19  
**范围：** 以 `I8888.XDCE` 铁矿石连续合约为信号源，输出可直接复制到聚宽的单品种 CTA 脚本。

## 背景与根因

原始策略是 20/60 日均线的多头/空仓系统。2024 年以后铁矿石进入震荡下行、政策驱动和供需偏宽松阶段，长多策略缺少正收益来源。原始代码还存在以下执行层风险：

- `get_bars` 未显式限定到前一交易日，信号截点不透明；
- 以月份后缀挑选合约，没有严格校验合约年份、上市状态和到期剩余天数；
- 以当月倒数第 5 个交易日换月，但铁矿石合约在交割月第 10 个交易日结束交易，存在持有到期合约的风险；
- 下单后直接更新 `g.flag`，没有按实际成交和实际持仓同步状态；
- 连续合约用于信号、具体月合约用于交易，换月价差和合约选择没有被明确控制；
- 手数只按账户净值和价格计算，没有同时约束 ATR 风险和保证金占用。

## 目标

1. 保留原始 20/60 趋势思想，先提供可比的长多/空仓模式。
2. 通过参数 `ALLOW_SHORT` 提供可选的双向趋势模式，使下跌趋势有收益来源。
3. 所有信号和下单价格只使用 `context.previous_date` 及更早数据。
4. 严格执行安全换月：先平旧合约，确认实际持仓归零后再开新合约。
5. 以 ATR 风险预算和最大保证金占用共同决定手数。
6. 保留原始手续费和滑点参数，避免通过降低交易成本制造回测收益。
7. 输出结构化日志，便于按年份、市场状态、换月、成交失败和空仓原因做归因。

## 方案选择

### 方案 A：稳健长多/空仓

只在确认的上行趋势中做多，确认下行趋势后空仓。优点是与原始策略最可比，风险边界清晰；缺点是 2024 年后的下行阶段只能减少损失，不能产生收益。

### 方案 B：双向趋势增强版（采用）

在方案 A 的信号和执行框架上增加空头方向，并使用更低的保证金上限和 ATR 风险预算。通过 `ALLOW_SHORT=False` 可以退化为方案 A。这样既保留原始逻辑的对照能力，也能验证 2024 年后的收益是否来自空头趋势，而不是来自回测口径变化。

### 方案 C：趋势 + 基差/跨期结构

加入近月/次近月价差、持仓量和成交量筛选。该方案可能改善合约选择，但数据字段和聚宽平台兼容性要求更高，不纳入本次首版。

## V2 行为设计

### 信号

- 信号源：`I8888.XDCE` 连续合约日线。
- 基础窗口：20 日均线和 60 日均线。
- 上行确认：收盘价高于 20 日均线、20 日均线高于 60 日均线、60 日均线斜率为正，并通过最小均线间距缓冲。
- 下行确认：对应条件全部反向。
- 连续确认：上行和下行均要求连续 2 个交易日确认。
- 震荡区：不新开仓；已有仓位保留到反向趋势确认或风险退出。
- 退出：反向趋势确认、ATR 紧急止损、合约换月失败保护或有效价格缺失时不盲目开新仓。

### 合约选择与换月

- 使用 `get_all_securities(['futures'], date=signal_date)` 获取点时合约列表。
- 只接受格式为 `I####.XDCE` 且上市、未到期的铁矿石合约。
- 剔除距离最后交易日不足 `roll_days_before_expiry` 的合约。
- 默认选择按到期日排序后的第二个合约，保持与原始策略的次远月倾向；只有一个合约可用时退化为第一个。
- 目标合约变化时，先平旧合约；只有实际持仓归零才允许下一交易日开新合约。

### 手数与风险

- 合约乘数固定为 100 吨/手，与原始 I 品种口径一致。
- 使用连续合约 20 日 ATR 估算每手一 ATR 的风险。
- `risk_budget = total_value * risk_per_atr`。
- 手数同时受风险预算和 `max_margin_usage` 限制，向下取整。
- 默认最大保证金占用 45%，低于原始版本的隐含风险暴露。
- 多空使用同一套手数公式，便于归因比较。

### 状态与日志

- 交易状态不再依赖单一 `g.flag`，而是以实际持仓为准。
- 记录信号日期、趋势状态、目标方向、目标合约、ATR、目标手数、换月动作、订单成交量和跳过原因。
- 订单部分成交时保留风险状态，不把计划手数误认为实际持仓。

## 验证要求

实现后必须完成：

1. Python 语法编译检查。
2. 用最小聚宽 API stub 测试信号确认、点时合约筛选、到期边界、先平后开和手数上限。
3. 静态检查不得出现原始的无截止日期 `get_bars`、无成交直接置位和跨品种合约匹配。
4. 聚宽平台至少分别回测 `ALLOW_SHORT=False` 与 `ALLOW_SHORT=True`。
5. 回测输出按年份拆分，重点比较 2024、2025 和 2026；同时记录收益、最大回撤、交易次数、换月次数、成交率和保证金峰值。

## 明确不做

- 不在本版本加入机器学习、基差预测或多品种组合。
- 不调整手续费和滑点以追求更高收益。
- 不把长多和双向模式混成不可解释的单一结果；脚本必须能通过一个参数切换并单独回测。
