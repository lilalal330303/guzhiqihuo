# 大小盘反复横跳 V2.2 原始兼容版设计

## 目标

以原始“干积分-大小盘反复横跳 V2.0”作为行为基线，恢复其收益特征和交易频率；只修复未来数据、异常数据和交易接口问题，不把 V2.1 的趋势、波动率、硬性空仓和历史指数成分池改动带入默认路径。

## 基线诊断

原始版的核心信号不是指数价格趋势，而是：

1. 获取国证2000（`399303.XSHE`）和中证500（`399905.XSHE`）成分股；
2. 计算每只成分股最近 20 日收益，再取横截面平均值；
3. 使用 `mean_2000 / mean_500 > 1.2` 作为分支条件；
4. 条件成立进入原始 `BIG` 选股分支，否则进入原始 `SMALL` 选股分支。

原始变量命名与指数含义存在反向命名，但 V2.2 必须保留其有效行为，不在默认版本中重命名或改变分支方向。V2.1 将该信号替换为指数价格收益、波动率标准化、60 日趋势确认、滞回和风险空仓，属于高扰动改动，已被回测证明显著弱于原始版。

## 设计

### 1. 信号层

- 默认 `signal_mode = "original_compatible"`。
- 默认窗口为 20 个交易日。
- 使用历史可用的成分股横截面收盘价收益。
- 使用 `mean_2000 / mean_500 > ratio_threshold`，默认 `ratio_threshold = 1.2`。
- 使用稳健数据函数处理空池、缺失值、零价格和样本不足；数据不足时跳过本次调仓，不改变持仓。
- 不默认加入指数均线、波动率标准化、趋势确认或 risk-off 空仓。

### 2. 选股层

- 保留原始全市场候选池和分支行为：`SMALL` 使用过滤后的 `choice`，`BIG` 的 B0 基线保留原始代码实际传入基本面查询的 `stocks`，不把“修正候选池”混入基线；两者都保留停牌、ST、科创/北交/创业板、次新和涨跌停处理代码，并记录各阶段数量。
- `SMALL` 分支保留原始 ROE、ROA、低价和小市值排序逻辑，默认 `max_price = 10`、`stock_num = 5`。
- `BIG` 分支保留原始估值、盈利能力、成长性和市值排序逻辑。
- 财务数据全部以 `context.previous_date` 作为 `date` 参数，避免原始 `date=None` 在回测中的信息时点不透明。
- 原始的“持仓中最近涨停股票暂不因排名变化卖出”规则保留。

### 3. 交易层

- 保留月初第一个交易日调仓、09:30 执行、等额分配现金。
- 默认 `FixedSlippage(0)`，同时保留参数化滑点，供后续成本敏感性测试使用。
- 交易接口使用 `order_target_value`，增加空订单、停牌、涨跌停和异常日志处理。
- 原始版本文件不覆盖；新脚本独立输出为 `reports/joinquant_size_style_rotation_v22_original_compatible.py`。

### 4. 实验开关

默认关闭以下选项，避免改变原始基线：

- `winsorize_returns = False`：可选对成分股收益做 5%/95% 截尾；
- `market_guard = False`：可选使用市场指数 60 日均线作为保护；
- `slippage = 0.0`：回测时单独比较 `0 / 0.001 / 0.002`；
- `use_historical_constituents = True`：默认使用信号日历史成分股，避免未来成分回填。

## 优化路径

按单变量、分阶段方式进行：

1. B0：原始行为兼容 + 历史日期修复 + 交易接口修复；
2. B1：只打开收益截尾，比较异常成分股对信号的影响；
3. B2：只打开市场保护，比较回撤与收益损失；
4. B3：分别测试滑点 `0 / 0.001 / 0.002`；
5. B4：在不同阶段和不同成本下比较收益、年化收益、最大回撤、交易次数、胜率、换手率和空调仓次数。

任何单项优化只有在全样本、早期样本、震荡样本和近期样本均未显著破坏基线时，才进入下一阶段。

`BIG` 从原始 `stocks` 改为过滤后的 `choice` 单独列为后续实验，不属于 B0。

## 数据与未来函数约束

- 所有行情信号截止到 `context.previous_date`。
- 所有财务查询显式传入 `date=context.previous_date`。
- 历史指数成分调用显式传入 `date=cutoff`。
- 信号在调仓日 09:30 产生并执行，禁止使用当日收盘后数据。

## 测试与验收

纯函数测试覆盖：

- 原始比值阈值的分支方向；
- 横截面平均收益的缺失值、零值和样本不足；
- 历史成分池与日期参数；
- B0 默认开关关闭全部新增风控；
- 目标持仓合并、涨停保护和等额资金分配。

静态检查覆盖：

- 脚本可被 Python AST 解析和编译；
- `get_fundamentals`、`get_index_stocks` 均显式使用历史 `date`；
- 不修改原始策略文件；
- 本地完整测试套件通过。

交付物：独立聚宽 Python3 脚本、回测对照说明、纯函数测试和优化矩阵说明。
