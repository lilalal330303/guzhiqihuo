# #39 大小盘反复横跳 V2.1 设计规格

## 目标

在保留原始策略有效的月频调仓、5只持仓、昨日涨停股处理和大小盘候选池框架的前提下，修复 V1 改变原始信号语义、风格频繁切换和交易摩擦过高的问题，产出一份可直接复制到聚宽回测的 V2.1 脚本。

## 现状诊断

原始脚本把国证2000的20日收益命名为 `change_BIG`，当国证2000相对中证500明显更强时选择 `BIG` 候选；否则选择 `SMALL` 候选。这实际上形成了“买近期相对落后风格”的反转规则。V1 改为将中证500定义为 BIG、国证2000定义为 SMALL，并在两者仅有微小差异时直接选择收益更高者，导致信号从反转切换为动量，并取消了原始的切换阈值。

V1 同时把 `FixedSlippage(0)` 改为 `PriceRelatedSlippage(0.002)`，并把调仓时间从 09:30 改为 09:35。V1 的历史日期基本面查询是正确的未来数据修复，应保留。聚宽官方文档支持使用历史日期查询基本面与历史指数成分股。

## 策略设计

### 1. 风格信号

- 风格标的：国证2000 `399303.XSHE` 与中证500 `399905.XSHE`。
- 直接使用指数收盘价计算信号，避免逐成分股平均收益受停牌、缺失值和样本变化影响。
- 20日信号采用反转方向，权重 0.65；60日信号采用趋势方向，权重 0.35。
- 各周期收益使用对应窗口波动率进行标准化，减少两个指数波动率不同带来的偏置。
- 仅当候选风格得分领先当前风格超过 `switch_gap` 且当前风格至少持有 `min_style_months` 个月时切换。
- 信号无法计算时保持当前风格，不因临时数据为空而清仓。

### 2. 市场防守

- 使用中证全指 `000985.XSHG` 作为市场状态参考。
- 当中证全指收盘低于60日均线、20日收益为负，且两个风格指数都低于60日均线时进入风险关闭状态。
- 风险关闭时目标持仓为空；恢复条件为中证全指重新站上60日均线且20日收益转正。
- 防守逻辑作为参数开关保留，默认开启，便于与原版/V1做对照回测。

### 3. 候选池和排序

- SMALL 候选限制在历史国证2000成分股；BIG 候选限制在历史中证500成分股。
- 统一使用 `context.previous_date` 查询基本面。
- 保留上市年限、停牌、ST、涨跌停和无效价格过滤。
- 默认关闭硬性10元价格上限，避免把低价偏好误当成小盘因子；保留可配置的 `max_price` 参数。
- 小盘池以质量过滤后的小市值排序；大盘池以质量和估值过滤后的市值排序。
- 先取 `stock_num * candidate_buffer` 个候选，调仓时优先保留仍处于候选缓冲区内的现有持仓，降低无效换手。

### 4. 交易执行

- 月初 09:35 调仓，先卖后买。
- 昨日涨停股继续沿用尾盘开板卖出逻辑。
- 默认滑点为 0.001，并作为参数保留；聚宽回测时同时测试 0、0.001、0.002 三档。
- 不新增个股止盈止损，避免把风格轮动、个股选择和退出规则混在一次迭代中。

## 数据安全与边界

- 所有历史行情和基本面信号只能使用 `context.previous_date` 及之前的数据。
- `get_current_data` 仅用于当日成交可行性检查，不参与历史排序或风格信号计算。
- 历史指数成分股通过 `get_index_stocks(index, date=cutoff)`获取。
- 交易失败、候选不足、数据为空时保持可执行的现有持仓，不强行补仓或清仓。

## 验证标准

本地验证：

1. Python AST 解析成功，且脚本不依赖本地 `jqdata` 导入即可完成静态解析。
2. 风格信号测试覆盖：反转方向、阈值滞回、最短持有期、空数据保持。
3. 市场防守测试覆盖：风险关闭、恢复和不满足条件时保持。
4. 候选池测试覆盖：历史成分限制、基本面日期、涨跌停与候选缓冲。
5. 静态扫描禁止 `get_fundamentals(..., date=None)`、未注明日期的指数成分调用和使用当前行情参与历史排序。

聚宽验证：

1. 与原始版、V1使用相同回测区间、初始资金和基准。
2. 比较总收益、年化收益、最大回撤、交易次数、胜率、换手率和滑点敏感性。
3. 分阶段比较 2020-2021、2022-2023、2024-2026，避免只优化全样本。
4. 重点记录风格切换日期、风险关闭区间、候选不足次数、订单成交率和无法卖出的涨跌停持仓。
