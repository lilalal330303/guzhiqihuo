# #39 大小盘反复横跳 V2.1

脚本：`reports/joinquant_size_style_rotation_v21.py`

## 这版修复了什么

V1 相比原始版改变了三件会直接影响收益的事：

1. 原始代码实际是“近期强风格上涨后，买相对落后风格”的反转规则；V1 改成了谁强买谁。
2. 原始版使用 `2000/500 > 1.2` 的切换阈值；V1 使用简单大小比较，容易在两种风格之间来回换仓。
3. 原始版是 0 滑点；V1 使用 0.2% 价格相关滑点，5只股票集中调仓时成本明显增加。

V2.1 默认恢复反转优先，同时加入60日趋势确认、最短持有期、风格领先幅度阈值、历史指数成分池、候选缓冲区和市场防守开关。

## 默认参数

| 参数 | 默认值 | 说明 |
|---|---:|---|
| `stock_num` | 5 | 最大持仓数量 |
| `candidate_buffer` | 2 | 候选缓冲倍数，默认保留前10名候选 |
| `switch_gap` | 0.10 | 风格得分领先幅度 |
| `min_style_months` | 2 | 风格最短持有月数 |
| `max_price` | 0 | 0表示关闭价格上限 |
| `risk_off_enabled` | True | 市场趋势转弱时进入现金 |
| `slippage` | 0.001 | 价格相关滑点，聚宽端应做敏感性测试 |

## 聚宽回测矩阵

所有比较应保持初始资金、基准、回测区间、手续费和调仓时间一致。

| 版本 | 风格信号 | 成分池 | 防守 | 滑点 |
|---|---|---|---|---:|
| 原始版 | 20日反转/原始阈值 | 全市场过滤 | 无 | 0 |
| V1 | 20日强者优先 | 全市场过滤 | 无 | 0.002 |
| V2.1 | 20日反转 + 60日趋势 | 历史2000/500成分 | 可开关 | 0/0.001/0.002 |

建议分别回测：

- 2020-01-01 至 2026-07-18：全样本
- 2020-01-01 至 2021-12-31：早期样本
- 2022-01-01 至 2023-12-31：震荡/弱市样本
- 2024-01-01 至 2026-07-18：近期样本

重点记录：总收益、年化收益、最大回撤、交易次数、胜率、换手率、滑点敏感性、风格切换日期、风险关闭区间、候选为空次数和订单成交率。

## 使用方式

将 `joinquant_size_style_rotation_v21.py` 全文复制到聚宽 Python3 策略编辑器，先用默认参数运行，再分别修改：

```python
"slippage": 0.0
"slippage": 0.001
"slippage": 0.002
```

并分别测试：

```python
"risk_off_enabled": True
"risk_off_enabled": False
```

不要只比较全样本总收益；如果 V2.1 只在全样本提升、但在三个分阶段中明显恶化，应视为过拟合信号。

## 聚宽端观察点

- 日志中确认 `SMALL/BIG` 风格得分、当前风格、持有月数和 `risk_off` 状态。
- 确认基本面查询均以 `context.previous_date` 为日期。
- 确认风险关闭时没有新买入，涨停锁仓在可卖出前不会被强制报错。
- 确认历史指数成分池调用为 `get_index_stocks(index, date=cutoff)`。
- 记录候选不足、订单未成交、涨跌停无法交易和实际成交数量。
