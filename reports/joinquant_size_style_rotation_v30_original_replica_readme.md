# V3.0 原始复刻：运行与实验说明

## 首次复核

1. 将 `joinquant_size_style_rotation_v30_original_replica.py` 复制到聚宽 Python3 策略编辑器。
2. 第一次对照时保持 `RUN_MODE = "ORIGINAL_REPLICA"`。
3. 使用原始截图/回测的日期范围、初始资金、基准和分钟频率。
4. 先比较每月分支、目标列表、交易次数、成交明细和日志，再比较收益率。
5. 只有基线对齐后，才将 `RUN_MODE = "STRICT_ASOF"` 作为 E1。

不要在用户实际运行脚本前声称任何云端回测收益、年化收益或其他云端指标。截图中的指标只能作为对照，不能当作新脚本的运行结果。

## 保留的基线设置

- 基准：`000985.XSHG`；启用实盘价格（`use_real_price=True`）和避免未来数据（`avoid_future_data=True`）。
- 滑点为零；保留原始交易成本；持股数量为 5。
- 风格窗口为 20 个交易日，近期涨停窗口为 40 个交易日。
- 每月第 1 个交易日调仓；任务时间为 09:05、14:00、14:30。

## 两种模式的边界

`ORIGINAL_REPLICA` 为提交基线的复现模式：有意保留基本面查询的 `date=None`，以及使用当前时间的指数成分查询，以便比较原始回测路径。因此它不是对实盘无未来函数安全性的证明。

`STRICT_ASOF` 只把上述两处日期语义改为 `context.previous_date`。信号、候选列表、分支判断、调度和下单规则全部保持一致。

## 明确拒绝的路径

本版本不加入 V2.1 趋势确认、波动率归一化、risk-off cash（`risk.off`）、滞后/迟滞、candidate buffer（`candidate.buffer`）、市场保护（market guard）、缩尾（winsorization）或强制保留持仓。由 `pivot_table(sort=...)` 引起的无交易问题，已通过形状兼容适配器修复，不是通过新增过滤器修复。

## 单变量实验顺序

- **E1**：比较 `ORIGINAL_REPLICA` 与 `STRICT_ASOF`。
- **E2**：选定信号路径后，仅测试滑点 `0`、`0.001`、`0.002`。
- **E3**：固定信号和成本后，比较每月第 1 个交易日与一个单独的替代调仓日。
- **E4**：固定调度后，比较 BIG 的宽 `stocks` 查询与筛选后的 `choice` 查询。

每个实验只改变一个变量，并必须同时报告全样本以及 2020-2021、2022-2023、2024-2026 三个窗口的：收益率、年化收益率、最大回撤、交易次数、胜率、换手率和空目标月份。先记录分支、目标、成交与日志，再解释收益变化。
