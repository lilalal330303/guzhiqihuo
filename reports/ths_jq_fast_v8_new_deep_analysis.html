4. 执行层规则：基本达成

两边完整输出 1575 条 WUFU_ORDER_PLAN，包含 estimated_cost / residual_cash / round_lot。本轮 order_fail 均为 0。权益差异主要来自弱市和目标错位后的复利路径分叉。

弱市根本统一方案

第一层：同步主线先恢复稳定

把同花顺 WEAK_MIN_VALID_INDEXES 从 3 改为 4。只要四个指数中任意一个缺失，立即触发 WUFU_WEAK_FALLBACK。A500 未修通前，收益对标主线继续使用回放弱市；自动弱市只作为诊断分支。

第二层：建立弱市日历 SSOT

本地每天生成统一弱市状态表：trade_date / is_weak / source / below_count / above_count / weak_start / weak_days / detail_hash，再嵌入同花顺和聚宽脚本。两端输出 WUFU_WEAK_CALENDAR_VERSION 和 WUFU_WEAK_SOURCE，保证三边共享同一事实源。

第三层：自动 A500 源修通后再切主线

自动源必须逐日输出四个指数的 resolved code、last_date、close、MA10、above/below/equal，以及状态机 before/after/weak_start/weak_days。只有当自动弱市匹配率达到 99%+，才替代弱市日历。

优化路径

  • V9：恢复同步主线。强制四指数有效，A500 缺失即 fallback，目标是弱市回到 95%+，目标回到 95%+。
  • V10：A500 数据源专项修复。尝试同花顺指数函数或板块指数函数，而不仅是证券行情 get_price。
  • V11：完全复刻弱市状态机。统一 previous_date、MA10 窗口、交易日 weak_days、进入退出优先级。
  • V12：弱市一致后，修复剩余池子/评分边界的 23 天。
