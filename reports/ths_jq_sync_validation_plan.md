# 同花顺 / 聚宽同步验证计划

## 本轮脚本

- 同花顺/SuperMind：`reports/同花顺_聚宽五福ETF同步版.py`
- 聚宽/JoinQuant：`reports/聚宽_五福ETF同步验证版.py`

两个脚本均已拆分为：

- 09:40：弱市状态、成交额阈值、动态池/合并池。
- 13:10：生成目标，输出 `WUFU_SIGNAL`。
- 13:11：执行下单，输出 `WUFU_EXECUTE`。
- 15:10：日内状态重置。

## 已修复的问题

同花顺脚本取消了无时间 `run_daily(func)` fallback。上一轮日志中所有交易都发生在 `09:31:00`，原因高度疑似为同花顺不接受 `run_daily(func, "13:10")` 后，脚本 fallback 到 `run_daily(func)`，平台默认在开盘第一根 bar 调用。

当前脚本只允许两种触发路径：

1. `run_daily(func, "13:10")` 注册成功。
2. 注册失败时只记录日志，实际由 `handle_bar()` 的 `hhmm >= 1310 / 1311` 时间门触发。

不会再注册无时间回调。

## 验收标准

新日志必须满足：

| 检查项 | 通过标准 |
| --- | --- |
| 同花顺信号时间 | `WUFU_SIGNAL now=... 13:10` 或之后，不能是 09:31 |
| 同花顺执行时间 | `WUFU_EXECUTE now=... 13:11` 或之后，不能是 09:31 |
| 聚宽信号时间 | `WUFU_SIGNAL now=... 13:10` |
| 聚宽执行时间 | `WUFU_EXECUTE now=... 13:11` |
| 每日信号数量 | 每个交易日最多 1 条 |
| 每日执行数量 | 每个交易日最多 1 条 |
| 弱市状态 | 同花顺与聚宽逐日一致或能解释差异 |
| 阈值 | 同花顺与聚宽同日数量级一致 |
| 目标 | 优先比较 `WUFU_SIGNAL target`，不是最终持仓 |

## 日线与分钟复验

1. 先跑同花顺分钟级回测。
2. 再跑同花顺日线回测。
3. 只有当两者都不再出现 09:31 交易后，才比较收益差异。

判断口径：

- 若日线仍远高于分钟，优先怀疑日线使用了全天成交量/收盘价，不能作为实盘近似。
- 若分钟与聚宽 13:10 目标接近，分钟版更适合作为实盘近似。
- 若同花顺分钟与聚宽目标仍大幅不同，下一步检查 `current_price`、分钟成交量、ETF 名称清洗和动态池。

## 下一轮日志解析字段

请重点导出或保留这些行：

- `WUFU_MORNING`
- `WUFU_SIGNAL`
- `WUFU_EXECUTE`
- `close value`
- 平台成交量限制 warning
- 订单无效 / 资金不足 warning

用这些行即可复刻逐日差异表：时间、弱市、阈值、池子大小、目标、执行目标、净值、成交容量限制。
