# 五福 ETF V8 迭代与本地回测分析

## 本轮目标

1. 将同花顺弱市状态从“聚宽日志回放”推进到“本地指数自动计算优先”。
2. 找出同花顺可用的中证 A500 数据源候选。
3. 继续保持弱市匹配率和目标匹配率 95% 以上。
4. 处理执行层的 100 份取整、碎股、资金不足、成本预留规则。
5. 输出新版同花顺脚本、聚宽脚本和本地执行版回测脚本，并完成一轮本地回测。

## 中证 A500 数据源结论

公开口径确认：

- 中证指数有限公司官网资料显示，中证 A500 指数代码为 `000510`。
- 同花顺行情中心的中证 A500 页面 URL 使用 `1B0510`，页面展示名称为“中证A500000510”。

因此 V8 同花顺脚本采用以下候选顺序探测：

`000510.SH -> 000510.XSHG -> 000510.CSI -> 000510 -> 1B0510 -> 1B0510.SH`

脚本运行时会输出 `WUFU_INDEX_RESOLVE`，用于确认实际在同花顺环境中哪个代码能返回足够日线。若自动指数源不足 3 个有效指数，则输出 `WUFU_WEAK_FALLBACK`，临时回退到 V7 的聚宽弱市区间，避免弱市匹配率掉下 95%。

## 平台脚本改动

### 同花顺 V8

- 默认启用自动弱市计算：`USE_EXTERNAL_WEAK_STATES = False`。
- 保留聚宽日志弱市兜底：`FALLBACK_EXTERNAL_WEAK_STATES = True`。
- 中证 A500 增加同花顺内部候选 `1B0510 / 1B0510.SH`。
- 初始化日志升级为 `WUFU_THS_FAST_V8`。
- 下单计划增加：
  - `estimated_cost`
  - `residual_cash`
  - `round_lot`
- 买入预算按 0.2% 现金缓冲、最低佣金、佣金和滑点预留后，再按 100 份向下取整。

### 聚宽 V8

- 初始化日志升级为 `WUFU_JQ_FIXED_POOL_V8`。
- 弱市指数继续使用 `000300.XSHG / 399101.XSHE / 399006.XSHE / 000510.XSHG`。
- 下单计划字段与同花顺一致。
- 买入预算同样加入成本预留和 100 份向下取整。

## 本地执行版回测

本地脚本：`reports/run_wufu_v8_local_execution_backtest.py`

本地回测规则：

- 时间：2020-01-02 到 2026-07-06。
- 初始资金：100 万。
- 信号：本地五福 ETF 目标生成。
- 弱市：本地 `000300 / 399101 / 399006 / 000510` 指数日线，聚宽风格状态机。
- 执行：T 日信号，T+1 日执行。
- 成本：ETF 佣金 0.01%、滑点 0.01%、最低佣金 5 元。
- 现金：保留 0.2% 缓冲。
- 份额：按 100 份向下取整。
- 数据处理：剔除单日涨跌幅绝对值超过 25% 的疑似断点标的；持仓估值使用前值填充，避免缺失价把权益打穿。

## 与 V7 平台验收目标的关系

V7 平台日志已经达到：

- 弱市匹配率：100%。
- 目标匹配率：98.48%。

V8 的设计不是强行改目标，而是在不破坏 95%+ 匹配率的前提下，完成两件事：

1. 让同花顺自动探测中证 A500 数据源，逐步摆脱弱市日志回放。
2. 让执行层规则更贴近真实下单，减少资金不足和碎股误差。

因此下一轮平台验收重点是：

- 同花顺日志中 `WUFU_INDEX_RESOLVE code=000510.SH resolved=... rows=...` 的实际 resolved 值。
- 是否出现 `WUFU_WEAK_FALLBACK`。
- 弱市匹配率是否仍保持 95% 以上，理想状态为 100%。
- 目标匹配率是否仍保持 95% 以上，理想状态不低于 V7 的 98.48%。
- `WUFU_ORDER_PLAN` 的 `estimated_cost / residual_cash / shares` 是否使资金不足类订单减少。
